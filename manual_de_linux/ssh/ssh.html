2. Copia de Archivos Remota

2.1 Copiar un archivo desde una máquina remota a la máquina local:

  • Sintaxis
scp usuario@equipo:/ruta/fichero_origen ruta/destino_local

Ejemplo:

scp antoniorodas@192.168.254.48:/home/antoniorodas/fichero.txt /tmp
Nota

En este ejemplo es desde la maquina "VMRodas.2" Hacia la maquina "VMRodas.1"

Explicación: La tabla explica cada componente del comando scp antoniorodas@192.168.254.44:/home/antoniorodas/fichero.txt /tmp.

Comando Descripción
scp Inicia el comando de Secure Copy Protocol.
antoniorodas@192.168.254.47: Especifica el usuario (antoniorodas) y la dirección IP del sistema remoto (192.168.254.47).
/home/antoniorodas/fichero.txt Es la ruta completa (Absoluta) del archivo en la máquina remota que se desea copiar. En este caso, es el archivo fichero.txt ubicado en el directorio /home/antoniorodas del sistema remoto.
/tmp Especifica el destino en la máquina local donde se copiará el archivo fichero.txt. En este caso, el archivo se copiará al directorio /tmp en la máquina local.

3. Túnel SSH: Redirigir Puertos

Instalación del Servidor Web

Antes de empezar, es necesario instalar un servicio que tenga una interfaz web. En este caso, utilizaremos Apache. Para instalarlo, ejecuta el siguiente comando:

sudo apt install apache2
Nota

Este manual asume que el servidor Apache se instalará en la máquina VMRodas.2 con la dirección IP 192.168.254.47. en vuestro caso la maquina y la ip que corresponda.

Establecer un Túnel SSH

Desde la máquina cliente, establece un túnel SSH con el siguiente comando:

ssh antoniorodas@192.168.254.47 -Nf -L 999:192.168.254.47:80

Sintaxis del Comando

ssh usuario@ipdelservidor -Nf -L puerto_local:ipdelservidor:puerto_remoto
  • -Nf: No abre una sesión en la máquina remota y el túnel permanece en segundo plano.
  • -L: Especifica la redirección de puertos local y remoto.
    Nota

    Si os da este error ejecutarlo como usuario privilegiado

Acceder al Servicio Web

Una vez establecido el túnel, accede a la interfaz web de Apache en tu navegador mediante una de las siguientes direcciones:

http://localhost:999

o

http://127.0.0.1:999

Comprobación de la Conexión con netstat

Para verificar que el túnel SSH está activo, utiliza el siguiente comando:

netstat -putona

Salida esperada:

tcp 0 0 127.0.0.1:999 0.0.0.0:* LISTEN 16709/ssh

Cierre del Túnel SSH (Maquina Cliente)

Para cerrar el túnel SSH, mata el proceso utilizando el comando kill -9 [PID], donde [PID] es el identificador del proceso que está utilizando el túnel SSH.

kill -9 [Numero del proceso]

Nota: Para encontrar el PID del proceso, puedes usar el comando ps aux | grep ssh.

5. Montaje de Directorios Remotos en Locales usando sshfs

  • Cliente: VMRodas.1 / 192.168.254.48
  • Servidor: VMRodas.2 / 192.168.254.47

5.1 Instalación en el Cliente

  1. Instalar sshfs: 
    sudo apt-get install sshfs
  2. Crear el punto de montaje:
    bash sudo mkdir -p /mnt/ssh

5.2 Conexión y Montaje

  1. Conectarse desde el cliente y montar el sistema de archivos remoto: 
    sudo sshfs root@192.168.254.47:/ /mnt/ssh
    Si todo ha salido bien, el servidor remoto estará montado en la ruta /mnt/ssh.

5.3 Desmontaje del Punto de Montaje

  1. Desmontar el punto de montaje utilizando umount: 
    sudo umount /mnt/ssh
    Alternativamente, puedes usar fusermount después de instalar fuse3: 
    sudo apt-get install fuse3
sudo fusermount -uz /mnt/ssh
