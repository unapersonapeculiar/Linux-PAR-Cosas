Manual de Utilidades SSH

1. Comandos Remotos (Sin Interaccion minimal con la maquina - Conectar Ejecutar y Cerrar )

  • Ejecutar un comando remoto:
ssh usuario@equipo comando

Ejemplo:

ssh antoniorodas@192.168.254.44 ps -ax

Nota: Cambiar ps -ax Por el comando que necesiteis

2. Copia de Archivos Remota

2.1 Copiar un archivo desde una máquina remota a la máquina local:

  • Sintaxis
scp usuario@equipo:/ruta/fichero_origen ruta/destino_local

Ejemplo:

scp antoniorodas@192.168.254.44:/home/antoniorodas/fichero.txt /tmp

Explicación: La tabla explica cada componente del comando scp antoniorodas@192.168.254.44:/home/antoniorodas/fichero.txt /tmp.

Comando Descripción
scp Inicia el comando de Secure Copy Protocol.
antoniorodas@192.168.254.44: Especifica el usuario (antoniorodas) y la dirección IP del sistema remoto (192.168.254.44).
/home/antoniorodas/fichero.txt Es la ruta completa (Absoluta) del archivo en la máquina remota que se desea copiar. En este caso, es el archivo fichero.txt ubicado en el directorio /home/antoniorodas del sistema remoto.
/tmp Especifica el destino en la máquina local donde se copiará el archivo fichero.txt. En este caso, el archivo se copiará al directorio /tmp en la máquina local.

2.2 Copiar un archivo desde la máquina local a una máquina remota:

  • Sintaxis 
    scp ruta/fichero_origen usuario@equipo:/ruta/destino_remoto
    Ejemplo: 
    scp /etc/hosts root@192.168.0.105:/etc

Explicación: La tabla explica cada componente del comando scp /etc/hosts root@192.168.0.105:/etc.

Comando Descripción
scp Inicia el comando de Secure Copy Protocol.
/etc/hosts Es la ruta completa del archivo en la máquina local que se desea copiar. En este caso, es el archivo hosts ubicado en el directorio /etc del sistema local.
root@192.168.0.105: Especifica el usuario (root) y la dirección IP del sistema remoto (192.168.0.105).
/etc Especifica el destino en la máquina remota donde se copiará el archivo hosts. En este caso, el archivo se copiará al directorio /etc en la máquina remota.

3. Túnel SSH: Redirigir Puertos

  • Establecer un túnel SSH:

    ssh usuario@192.168.1.32 -Nf -L 999:192.168.1.32:80

    El parámetro -Nf es para que no se abra una sesión en la máquina remota y permanezca en segundo plano.

    Accede de forma segura en tu navegador tecleando:

    http://localhost:999

  • Comprobación de la conexión con netstat:

    netstat -putona

    Salida esperada:

    tcp 0 0 127.0.0.1:999 0.0.0.0:* LISTEN 16709/ssh

4. Exportación Automática de Claves

  • Generar y exportar claves SSH:
    • Generación de claves: 
      ssh-keygen
    • Copia de claves: 
      ssh-copy-id usuario@192.168.1.105

5. Montaje de Directorios Remotos en Locales usando sshfs

  • Instalación en el cliente: 
    sudo apt-get install sshfs
  • Crear punto de montaje y conectar desde el cliente: 
    sudo mkdir /mnt/ssh
sudo sshfs root@192.168.1.147:/ /mnt/ssh
  • Desmontar el punto de montaje: 
    sudo umount /mnt/ssh
  • Montaje permanente desde FSTAB: 
    sudo nano /etc/fstab
    Añadir la línea: 
    sshfs#root@192.168.1.147:/ /mnt/ssh fuse defaults 0 0
  • Montaje automático con claves exportadas: 
    sudo mount -a

6. Conexiones de Túnel SSH

  • Objetivo:
    Establecer un canal de comunicación seguro entre dos máquinas.

  • Instalaciones:

    sudo apt-get install ssh

    Configurar en /etc/ssh/sshd_config:

    PermitTunnel yes

    Reiniciar el servicio:

    sudo systemctl restart ssh

  • Puesta en marcha del servicio:

    echo 1 | sudo tee /proc/sys/net/ipv4/ip_forward
sudo ssh -NTCf -w 0:0 usuario@192.168.1.105

  • Configuración del servicio:

    • Máquina B: 
      sudo ifconfig tun0 192.168.0.2 pointopoint 192.168.0.1
    • Máquina A: 
      sudo ifconfig tun0 192.168.0.1 pointopoint 192.168.0.2

  • Habilitar la ruta de la red:

    • En la máquina B: 
      sudo route add -net 192.168.0.0 netmask 255.255.255.0 gw 192.168.0.1 tun0
sudo route add -net 0.0.0.0 netmask 0.0.0.0 gw 192.168.0.1 tun0
    • En la máquina A: 
      sudo arp -sD 192.168.0.2 eth0 pub
